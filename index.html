品牌

品牌概览

Happycapy 是一台完全运行在浏览器中的 Agent 原生电脑。无需安装、无需配置终端，只要打开一个浏览器标签页，就能拥有一台功能完整的 AI 电脑：写代码、处理文档、生成图片、分析数据，并全天候替你执行任务。

它由 Claude Code 驱动，可调用 150+ 个 AI 模型。但重点不在技术栈，而在于：即使你不是开发者，它也真的好用。

Happycapy 曾登顶 Product Hunt 产品周榜与月榜第一。

品牌名书写规范：请始终写作 Happycapy —— 仅首字母 "H" 大写。"c" 不要大写，"happy" 和 "capy" 之间也不要加空格。

推广活动

推广目标

目标：在创作者、营销人和关注效率的受众中提升认知并带动注册——他们听说过 AI 工具，却常因复杂的上手门槛而被挡在门外。

我们不想要的：又一条"AI 改变一切"式的泛泛而谈。我们想要的是展示真实用例的内容——达人真正用 Happycapy 做过的事，哪怕很小、很具体。
受众

目标受众

那些对 AI 好奇却已被工具搞得疲惫的人。他们用过 ChatGPT，也许还试过 Notion AI。他们希望 AI 真正干活，而不只是聊天，也不想为此去学编程。

核心受众：创作者、自由职业者、小企业主、营销人、学生及其他专业消费级用户。

AI 探索者 内容创作者 / 达人 中小企业管理者 开发者
描述 希望在日常工作中直接用上 AI、而不必先搞懂底层原理的非技术型职场人。 活跃在 Instagram 和 TikTok 的创作者，需要以超出常人极限的速度产出精良内容。 需要简单而高效的内部系统，又不想雇开发者或购买昂贵的 SaaS 工具。 希望更快做原型、更快上线，而无需管理基础设施或拼接各种 API 的开发者。
典型场景 制作演示文稿、开展市场调研 制作视频、监控播放量与点赞、回复评论、内容分析 搭建数据看板、监控财务系统 搭建内部工具、自动化流水线、测试集成
痛点 工具过载、选择疲劳；上手门槛太技术化；难以融入真实工作流 缺乏技术能力；定制方案成本高；时间紧张；难以规模化 负担不起定制软件；现有工具过于复杂或定价过高；需要以最少的技术配置快速上线 样板化配置太多；API 管理负担重；单人原型迭代周期慢
产品

核心功能

Agent 原生体验

从底层为 Agent 式交互而设计。你不是在给聊天机器人写提示词，而是把任务委托给一个 Agent：它接下真实任务、拆解步骤、逐步执行，并向你汇报结果。

7×24 小时自动化运行

你不在线时，Happycapy 也在持续工作。安排定时任务、启动长时间运行的工作流，无论电脑合上还是你已入睡，醒来就能收到完成的成果。

内置技能库

30,000+ 个专用技能随取随用，覆盖内容生成、编程、分析、翻译、摘要、头脑风暴等场景，无需任何提示词工程。

云端沙箱执行

把复杂任务交给云端。在桌面与移动端无缝使用 Happycapy，文件始终保持一致，实现真正的全天候执行；本地文件不被改动、安全无虞。

Capy 邮箱

你的 Agent 拥有自己的邮箱地址。让 Happycapy 发送带收件人、附件和精美排版的邮件；他人也可以直接发邮件到你的 Capy 地址来触发工作流。

发布与分享

分享完整会话或单个文件，也可以把项目发布出去，让他人基于它二次创作并继续构建。

150+ 内置模型

Claude Opus 4.8、Opus 4.6、Sonnet 4.6、面向 Agent 工作流的 MiniMax M3 等 150+ 个模型，Agent 可直接调用，无需单独配置 API Key。

连接我的 Mac

一条命令即可连接本地 Mac。Agent 可以直接在你的电脑上执行命令、读写文件、处理任务，让云端自动化与桌面环境无缝衔接。

创作准则

最佳实践

✅ 建议 ❌ 避免
真人出镜 不要使用 AI 虚拟形象或遮挡面部；不要只播放广告视频
展示用 Happycapy 实际构建的真实用例 不要只停留在抽象、含糊的描述上
保持真实，呈现你自己的工作流或项目 不要显得照本宣科、机械生硬
突出用 Happycapy 构建有多简单、多快 不要把重点放在非核心功能上
用自然的语言讲清 Happycapy 解决了什么问题 不要把 Happycapy 说成一个普通的 AI 工具
分享你的创作过程，以及你个人喜欢它的哪些地方 不要过度承诺，也不要夸大或歪曲产品能力
鼓励你的观众亲自上手试一试 不要硬性推销，也不要塞入无关的行动号召（CTA）
用吸睛的开场钩子在第一时间抓住注意力：激发好奇、突出创新、给出可感知的价值。优先使用你过往验证过、观众反响好的钩子。 不要使用强行推销式的钩子，避免听起来太像广告的开场。
